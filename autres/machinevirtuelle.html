Qu'est ce qu'une machines virtuelle ?

Une machine virtuelle (VM) est un environnement informatique émulé qui fonctionne comme un ordinateur indépendant. Elle repose sur un logiciel appelé hyperviseur (comme VirtualBox, VMware, ou Hyper-V) qui permet de partager les ressources physiques d’un ordinateur (CPU, mémoire, stockage, etc.) entre plusieurs systèmes virtuels. Ces systèmes peuvent avoir leurs propres systèmes d'exploitation, configurations et applications, tout en restant isolés les uns des autres. À quoi sert une machine virtuelle ?

  • Tester différents systèmes d'exploitation/li>
  • Développement et test de logiciels
  • Isolation et sécurité
  • Consolidation des serveurs
  • Récupération et sauvegarde

En résumé, une machine virtuelle est un outil flexible et puissant, utilisé dans de nombreux domaines, que ce soit pour le développement, la formation, l'administration des systèmes ou l'optimisation des ressources.

Installation DNS

Qu'est-ce qu'un DNS primaire ?

Le DNS primaire (ou serveur DNS primaire) est un serveur qui contient l'enregistrement principal d'une zone de noms de domaine. Il est responsable de la gestion et de la distribution des informations sur les noms de domaine et leurs adresses IP associées.


À quoi ça sert ?

Le DNS (Domain Name System) est un système qui traduit les noms de domaine (ex : www.google.com) en adresses IP compréhensibles par les machines (ex : 142.250.185.14).

Un DNS primaire joue un rôle crucial dans ce processus en :
- Stockant les enregistrements DNS d’un domaine (A, CNAME, MX, TXT, etc.).
- Répondant aux requêtes DNS des utilisateurs pour diriger le trafic vers les bons serveurs.
- Synchronisant les données avec un DNS secondaire pour assurer la redondance et éviter les pannes.


Différence entre DNS primaire et secondaire

DNS primaire : Il détient la version "maître" des enregistrements DNS et peut être modifié directement.
DNS secondaire : Il est une copie du DNS primaire et sert de sauvegarde en cas de défaillance du primaire.


Exemple d'utilisation

Si tu tapes www.example.com dans ton navigateur :
1- Ton appareil envoie une requête au serveur DNS primaire du domaine.
2- Le serveur DNS retourne l'adresse IP correspondante.
3- Ton navigateur se connecte à cette adresse IP et affiche le site.

DHCP & DHCP failover

Qu'est-ce qu'un DHCP ?

Le DHCP (Dynamic Host Configuration Protocol) est un protocole réseau qui attribue automatiquement des adresses IP et d’autres paramètres réseau (passerelle, DNS, etc.) aux appareils connectés à un réseau.

À quoi ça sert ?

Le DHCP simplifie la gestion des adresses IP en :
1- Évitant la configuration manuelle de chaque appareil.
2- Réduisant les conflits d’adresses IP en attribuant des IP uniques.
3- Fournissant automatiquement des paramètres réseau comme la passerelle et les DNS.
4- Facilitant l’administration réseau grâce à une gestion centralisée des adresses IP.

Comment fonctionne un DHCP ?

Lorsqu’un appareil rejoint un réseau :
1- Il envoie une requête DHCP pour demander une adresse IP.
2- Le serveur DHCP attribue une adresse IP et d’autres paramètres réseau.
3- L’appareil utilise ces informations pour communiquer sur le réseau.
4- L’adresse IP est temporaire et peut être renouvelée avant son expiration.

Qu’est-ce que le DHCP Failover ?

Le DHCP Failover est une configuration permettant à deux serveurs DHCP de fonctionner ensemble pour assurer une haute disponibilité et éviter les pannes.

À quoi ça sert ?

Un DHCP Failover garantit :
1- Une tolérance aux pannes : Un second serveur DHCP prend le relais si le premier tombe en panne.
2- Une répartition de charge : Les adresses IP peuvent être distribuées par deux serveurs pour améliorer les performances.
3- Une continuité de service : Les appareils obtiennent toujours une adresse IP, même en cas de problème réseau.

Modes de fonctionnement du DHCP Failover

- Mode Load Balance : Les deux serveurs DHCP distribuent les adresses IP simultanément (ex: 50% chacun).
- Mode Hot Standby : Un serveur est actif et l’autre est en attente. Si le premier tombe en panne, le second prend automatiquement le relais.

Apache

Qu'est-ce qu'Apache ?

Apache HTTP Server (souvent appelé Apache) est un serveur web open-source qui permet d’héberger et de diffuser des sites web sur Internet ou un réseau local. Il est l’un des serveurs web les plus utilisés au monde.Apache HTTP Server (souvent appelé Apache) est un serveur web open-source qui permet d’héberger et de diffuser des sites web sur Internet ou un réseau local. Il est l’un des serveurs web les plus utilisés au monde.

À quoi ça sert ?

Un serveur Apache joue un rôle clé dans l’hébergement de sites web en :
1- Gérant les requêtes HTTP envoyées par les navigateurs (Chrome, Firefox, etc.).
2- Servant des fichiers HTML, CSS, JavaScript et autres ressources aux visiteurs.v
3- Permettant l’exécution de scripts côté serveur (PHP, Python, Perl, etc.).
4- Offrant une gestion avancée des accès et des permissions (authentification, restriction d’accès).
5- Supportant des modules supplémentaires pour améliorer ses fonctionnalités (sécurité, cache, compression, etc.).

Comment fonctionne Apache ?

1- Un utilisateur entre une URL dans son navigateur (ex: www.example.com).
2- Le navigateur envoie une requête HTTP au serveur Apache.
3- Apache traite la requête et recherche la page demandée.
4- Il renvoie la page web au navigateur, qui l’affiche à l’utilisateur.
